Getting Help

Try the following steps if you are having problems with SLURM:

  1. See if the problem is addressed in the SLURM FAQ, SLURM Troubleshooting Guide or the slurm-dev mailing list archive (Hosted by www.gmane.org).
  2. For run-time problems, try running the command or daemons in verbose mode (-v option), and see if additional information helps you resolve the problem.
  3. Customers of companies providing commercial support for SLURM should contact their support staff.
  4. Send a detailed description of the problem, the output from the command "scontrol show config", logs, back traces from any core files, etc. to slurm-dev@schedmd.com.
